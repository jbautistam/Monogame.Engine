### Opción 1: Funciones de Easing (Recomendada)

En lugar de una curva compleja, usamos funciones matemáticas ligeras para interpolar valores. Esto es el estándar en la industria hoy en día.

```csharp
using System;

public static class EasingFunctions
{
    // Suavizado cuadrático (entrada y salida)
    public static float InOutQuad(float t)
    {
        return t < 0.5f ? 2 * t * t : -1 + (4 - 2 * t) * t;
    }

    // Suavizado senoidal (muy natural para desvanecimientos)
    public static float InOutSine(float t)
    {
        return -(float)Math.Cos(Math.PI * t) / 2 + 0.5f;
    }

    // Suavizado cúbico (más dramático)
    public static float InOutCubic(float t)
    {
        return t < 0.5f ? 4 * t * t * t : (t - 1) * (2 * t - 2) * (2 * t - 2) + 1;
    }
}
```

**Uso en tu modificador:**

```csharp
public class ScaleOverLifetimeModifier : IParticleModifier
{
    private float _startScale;
    private float _endScale;
    private Func<float, float> _easingFunction; // Aquí usamos la función

    public ScaleOverLifetimeModifier(float start, float end, Func<float, float> easing = null)
    {
        _startScale = start;
        _endScale = end;
        _easingFunction = easing ?? EasingFunctions.InOutQuad; // Default
    }

    public void Modify(Particle particle, float deltaTime, float normalizedAge)
    {
        // Aplicamos la función de easing al porcentaje de vida (0 a 1)
        float easeValue = _easingFunction(normalizedAge);
        particle.Scale = MathHelper.Lerp(_startScale, _endScale, easeValue);
    }
}
```

**Ventaja:** Es extremadamente rápido (solo matemáticas básicas) y no depende de ninguna clase externa de MonoGame.

### Opción 2: Tu propia clase `FloatCurve` (Ligera)

Si realmente necesitas keyframes (puntos de control) como los que ofrecía `Curve`, crea tu propia versión simplificada optimizada para partículas.

```csharp
public struct Keyframe
{
    public float Time; // 0 a 1
    public float Value;
    public Keyframe(float time, float value) { Time = time; Value = value; }
}

public class FloatCurve
{
    private List<Keyframe> _keys;

    public FloatCurve()
    {
        _keys = new List<Keyframe>();
    }

    public void AddKey(float time, float value)
    {
        _keys.Add(new Keyframe(MathHelper.Clamp(time, 0, 1), value));
        // Ordenar por tiempo es importante
        _keys.Sort((a, b) => a.Time.CompareTo(b.Time));
    }

    public float Evaluate(float t)
    {
        if (_keys.Count == 0) return 0;
        if (_keys.Count == 1) return _keys[0].Value;
        if (t <= _keys[0].Time) return _keys[0].Value;
        if (t >= _keys[_keys.Count - 1].Time) return _keys[_keys.Count - 1].Value;

        // Encontrar segmento
        for (int i = 0; i < _keys.Count - 1; i++)
        {
            if (t >= _keys[i].Time && t <= _keys[i + 1].Time)
            {
                float localT = (t - _keys[i].Time) / (_keys[i + 1].Time - _keys[i].Time);
                return MathHelper.Lerp(_keys[i].Value, _keys[i + 1].Value, localT);
            }
        }
        return 0;
    }
}
```

## 1. El Problema: Draw Calls vs. Batching

Cada vez que llamas a `spriteBatch.Draw()`, MonoGame prepara datos para la GPU.

| Método | Draw Calls para 10,000 partículas | FPS Estimado |
|--------|-----------------------------------|--------------|
| `spriteBatch.Draw()` individual | 10,000 | 10-20 FPS |
| `SpriteBatch` con misma textura | 1 (automático) | 40-50 FPS |
| **VertexBuffer Custom** | **1 (manual)** | **60+ FPS** |

## 2. Nivel 1: Optimización Básica con SpriteBatch

Antes de crear un sistema complejo, asegúrate de que `SpriteBatch` pueda trabajar eficientemente.

### Reglas de Oro para SpriteBatch

1.  **Misma Textura:** Todas las partículas deben usar la **misma textura** para que `SpriteBatch` las agrupe automáticamente.
2.  **Begin/End Único:** Llama a `Begin()` una vez, dibuja todo, y llama a `End()` una vez.
3.  **Sin Cambios de Estado:** No cambies de textura o `Effect` entre partículas.

```csharp
// ❌ MAL: Begin/End dentro del bucle
foreach (var particle in particles)
{
    spriteBatch.Begin();
    spriteBatch.Draw(texture, particle.Position, particle.Color);
    spriteBatch.End();
}

// ✅ BIEN: Batch automático (si usan la misma textura)
spriteBatch.Begin(SpriteSortMode.Texture, BlendState.AlphaBlend);
foreach (var particle in particles)
{
    if (particle.IsActive)
        spriteBatch.Draw(texture, particle.Position, particle.Color);
}
spriteBatch.End();
```

**Límite:** Esto funciona bien hasta ~5,000 partículas. Después, el overhead de C# de cada llamada `Draw()` empieza a pesar.

## 3. Nivel 2: Sistema Profesional con VertexBuffer

Para miles de partículas, debemos evitar `spriteBatch.Draw()` por completo. Crearemos un **VertexBuffer** que contiene la geometría de todas las partículas y la enviaremos a la GPU en **una sola llamada**.

### A. Estructura de Vértice para Partículas

Necesitamos definir qué datos enviamos a la GPU por cada vértice. Usaremos `VertexPositionColorTexture` que ya está definido en MonoGame, pero configurado para 2D.

```csharp
using Microsoft.Xna.Framework;
using Microsoft.Xna.Framework.Graphics;

public struct ParticleVertex
{
    public Vector3 Position;    // XYZ (Z = 0 para 2D)
    public Color Color;
    public Vector2 TextureCoord; // UV

    public static readonly VertexDeclaration VertexDeclaration = new VertexDeclaration(
        new VertexElement(0, VertexElementFormat.Vector3, VertexElementUsage.Position, 0),
        new VertexElement(12, VertexElementFormat.Color, VertexElementUsage.Color, 0),
        new VertexElement(16, VertexElementFormat.Vector2, VertexElementUsage.TextureCoordinate, 0)
    );
}
```

### B. El Renderizador de Partículas (`ParticleRenderer`)

Esta clase gestiona el buffer y dibuja todo de una vez.

```csharp
public class ParticleRenderer
{
    private GraphicsDevice _graphicsDevice;
    private VertexBuffer _vertexBuffer;
    private IndexBuffer _indexBuffer;
    private BasicEffect _effect;
    private int _maxParticles;
    private ParticleVertex[] _vertexArray; // CPU-side array

    public ParticleRenderer(GraphicsDevice graphicsDevice, int maxParticles = 10000)
    {
        _graphicsDevice = graphicsDevice;
        _maxParticles = maxParticles;
        
        // 1. Crear array de vértices en CPU
        _vertexArray = new ParticleVertex[maxParticles * 4]; // 4 vértices por partícula (quad)

        // 2. Crear VertexBuffer en GPU
        _vertexBuffer = new VertexBuffer(
            graphicsDevice, 
            ParticleVertex.VertexDeclaration, 
            _vertexArray.Length, 
            BufferUsage.WriteOnly
        );

        // 3. Crear IndexBuffer (índices reutilizables)
        // Cada partícula son 2 triángulos (6 índices)
        ushort[] indices = new ushort[maxParticles * 6];
        for (int i = 0; i < maxParticles; i++)
        {
            int baseIndex = i * 4;
            int baseIdx = i * 6;
            
            indices[baseIdx + 0] = (ushort)(baseIndex + 0);
            indices[baseIdx + 1] = (ushort)(baseIndex + 1);
            indices[baseIdx + 2] = (ushort)(baseIndex + 2);
            indices[baseIdx + 3] = (ushort)(baseIndex + 2);
            indices[baseIdx + 4] = (ushort)(baseIndex + 3);
            indices[baseIdx + 5] = (ushort)(baseIndex + 0);
        }

        _indexBuffer = new IndexBuffer(
            graphicsDevice, 
            IndexElementSize.SixteenBits, 
            indices.Length, 
            BufferUsage.WriteOnly
        );
        _indexBuffer.SetData(indices);

        // 4. Configurar BasicEffect para 2D ortográfico
        _effect = new BasicEffect(graphicsDevice);
        _effect.TextureEnabled = true;
        _effect.VertexColorEnabled = true;
    }

    public void UpdateVertexBuffer(Particle[] particles, Texture2D texture)
    {
        int vertexIndex = 0;
        int activeCount = 0;

        foreach (var p in particles)
        {
            if (!p.IsActive) continue;

            // Calcular los 4 vértices del quad (centrado en la partícula)
            float halfSize = (texture.Width * p.Scale) / 2f;
            
            // Vértices: Top-Left, Top-Right, Bottom-Right, Bottom-Left
            _vertexArray[vertexIndex++] = new ParticleVertex
            {
                Position = new Vector3(p.Position.X - halfSize, p.Position.Y - halfSize, 0),
                Color = p.Color,
                TextureCoord = new Vector2(0, 0)
            };
            _vertexArray[vertexIndex++] = new ParticleVertex
            {
                Position = new Vector3(p.Position.X + halfSize, p.Position.Y - halfSize, 0),
                Color = p.Color,
                TextureCoord = new Vector2(1, 0)
            };
            _vertexArray[vertexIndex++] = new ParticleVertex
            {
                Position = new Vector3(p.Position.X + halfSize, p.Position.Y + halfSize, 0),
                Color = p.Color,
                TextureCoord = new Vector2(1, 1)
            };
            _vertexArray[vertexIndex++] = new ParticleVertex
            {
                Position = new Vector3(p.Position.X - halfSize, p.Position.Y + halfSize, 0),
                Color = p.Color,
                TextureCoord = new Vector2(0, 1)
            };

            activeCount++;
        }

        // Subir datos a la GPU
        _vertexBuffer.SetData(_vertexArray, 0, vertexIndex, SetDataOptions.Discard);
    }

    public void Render(Texture2D texture, Matrix projectionMatrix)
    {
        // Contar partículas activas para dibujar solo las necesarias
        int activeParticles = 0;
        for (int i = 0; i < _vertexArray.Length; i += 4)
        {
            if (_vertexArray[i].Color.A > 0) activeParticles++;
        }

        if (activeParticles == 0) return;

        _graphicsDevice.SetVertexBuffer(_vertexBuffer);
        _graphicsDevice.Indices = _indexBuffer;

        _effect.Texture = texture;
        _effect.Projection = projectionMatrix;

        foreach (var pass in _effect.CurrentTechnique.Passes)
        {
            pass.Apply();
            _graphicsDevice.DrawIndexedPrimitives(
                PrimitiveType.TriangleList,
                0,
                0,
                activeParticles * 4,
                0,
                activeParticles * 2
            );
        }
    }

    public void Dispose()
    {
        _vertexBuffer?.Dispose();
        _indexBuffer?.Dispose();
        _effect?.Dispose();
    }
}
```

## 5. Soporte para Rotación (Opcional pero Recomendado)

El código anterior no soporta rotación. Para añadirla, necesitamos calcular los vértices rotados:

```csharp
// Dentro de UpdateVertexBuffer, al calcular vértices:
float cos = (float)Math.Cos(p.Rotation);
float sin = (float)Math.Sin(p.Rotation);

// Vector desde el centro a cada esquina
Vector2[] corners = new Vector2[]
{
    new Vector2(-halfSize, -halfSize),
    new Vector2(halfSize, -halfSize),
    new Vector2(halfSize, halfSize),
    new Vector2(-halfSize, halfSize)
};

for (int i = 0; i < 4; i++)
{
    // Rotar el corner
    Vector2 rotated = new Vector2(
        corners[i].X * cos - corners[i].Y * sin,
        corners[i].X * sin + corners[i].Y * cos
    );

    _vertexArray[vertexIndex++] = new ParticleVertex
    {
        Position = new Vector3(p.Position + rotated, 0),
        Color = p.Color,
        TextureCoord = new Vector2(i % 2, i / 2) // UVs
    };
}
```

**Optimización:** Para evitar `Math.Cos/Sin` en cada partícula, usa las **Tablas LUT** que vimos antes:

## 1. Estructura de Animación (`SpriteAnimation`)

Primero, definimos cómo está organizada tu hoja de sprites (filas, columnas, FPS).

```csharp
using Microsoft.Xna.Framework;

public class SpriteAnimation
{
    public int TotalFrames { get; set; }
    public int Rows { get; set; }
    public int Columns { get; set; }
    public float FramesPerSecond { get; set; }
    public float FrameDuration => 1f / FramesPerSecond;

    public SpriteAnimation(int rows, int columns, float fps = 30f)
    {
        Rows = rows;
        Columns = columns;
        TotalFrames = rows * columns;
        FramesPerSecond = fps;
    }

    // Calcular UVs para un frame específico
    public RectangleF GetUVsForFrame(int frame)
    {
        frame = frame % TotalFrames;
        int row = frame / Columns;
        int col = frame % Columns;

        float uWidth = 1f / Columns;
        float vHeight = 1f / Rows;

        return new RectangleF(
            col * uWidth,
            row * vHeight,
            uWidth,
            vHeight
        );
    }
}

// Struct auxiliar para UVs (float es más preciso que int para UVs)
public struct RectangleF
{
    public float X, Y, Width, Height;
    public RectangleF(float x, float y, float w, float h) { X = x; Y = y; Width = w; Height = h; }
}
```

## 2. Modificación de la Clase `Particle`

Cada partícula necesita saber en qué frame de animación está.

```csharp
public class Particle
{
    // ... (propiedades anteriores: Position, Velocity, Color, etc.)

    // NUEVO: Propiedades de animación
    public int CurrentFrame { get; set; }
    public float FrameTimer { get; set; }
    public SpriteAnimation Animation { get; set; }
    public bool IsAnimated { get; set; }

    public void ResetAnimation()
    {
        CurrentFrame = 0;
        FrameTimer = 0;
    }

    public void UpdateAnimation(float deltaTime)
    {
        if (!IsAnimated || Animation == null) return;

        FrameTimer += deltaTime;
        if (FrameTimer >= Animation.FrameDuration)
        {
            FrameTimer -= Animation.FrameDuration;
            CurrentFrame = (CurrentFrame + 1) % Animation.TotalFrames;
        }
    }
}
```

## 3. Modificación del `ParticleRenderer` para Soportar UVs Animadas

Aquí está el cambio más importante. En lugar de UVs fijas (0,0 a 1,1), calculamos las UVs basadas en el frame actual.

```csharp
public class ParticleRenderer
{
    // ... (código anterior del renderer)

    public void UpdateVertexBuffer(Particle[] particles, Texture2D texture)
    {
        int vertexIndex = 0;

        foreach (var p in particles)
        {
            if (!p.IsActive) continue;

            // 1. Obtener UVs del frame actual
            RectangleF uvRect = new RectangleF(0, 0, 1, 1); // Default: textura completa
            
            if (p.IsAnimated && p.Animation != null)
            {
                uvRect = p.Animation.GetUVsForFrame(p.CurrentFrame);
            }

            // 2. Calcular tamaño de la partícula
            float halfSize = (texture.Width * p.Scale) / 2f;

            // 3. Calcular rotación (usando LUT para rendimiento)
            float cos = 1f, sin = 0f;
            if (p.Rotation != 0)
            {
                Vector2 sinCos = SinCosLUT.GetSinCosFast(p.Rotation);
                cos = sinCos.X;
                sin = sinCos.Y;
            }

            // 4. Definir las 4 esquinas del quad (antes de rotar)
            Vector2[] corners = new Vector2[]
            {
                new Vector2(-halfSize, -halfSize), // Top-Left
                new Vector2(halfSize, -halfSize),  // Top-Right
                new Vector2(halfSize, halfSize),   // Bottom-Right
                new Vector2(-halfSize, halfSize)   // Bottom-Left
            };

            // 5. UVs para cada esquina
            Vector2[] uvs = new Vector2[]
            {
                new Vector2(uvRect.X, uvRect.Y),                         // Top-Left
                new Vector2(uvRect.X + uvRect.Width, uvRect.Y),          // Top-Right
                new Vector2(uvRect.X + uvRect.Width, uvRect.Y + uvRect.Height), // Bottom-Right
                new Vector2(uvRect.X, uvRect.Y + uvRect.Height)          // Bottom-Left
            };

            // 6. Crear los 4 vértices con rotación aplicada
            for (int i = 0; i < 4; i++)
            {
                // Rotar el corner alrededor del centro
                Vector2 rotated = new Vector2(
                    corners[i].X * cos - corners[i].Y * sin,
                    corners[i].X * sin + corners[i].Y * cos
                );

                _vertexArray[vertexIndex++] = new ParticleVertex
                {
                    Position = new Vector3(p.Position + rotated, 0),
                    Color = p.Color,
                    TextureCoord = uvs[i]
                };
            }
        }

        // Subir datos a la GPU
        _vertexBuffer.SetData(_vertexArray, 0, vertexIndex, SetDataOptions.Discard);
    }
}
```

## 6. Actualización del `ParticleSystem`

Las partículas deben actualizar su animación en cada frame.

```csharp
public class ParticleSystem
{
    private Particle[] _particles;
    private int _maxParticles;

    public ParticleSystem(int maxParticles)
    {
        _maxParticles = maxParticles;
        _particles = new Particle[maxParticles];
        
        for (int i = 0; i < maxParticles; i++)
        {
            _particles[i] = new Particle();
        }
    }

    public void Update(GameTime gameTime)
    {
        float deltaTime = (float)gameTime.ElapsedGameTime.TotalSeconds;

        for (int i = 0; i < _maxParticles; i++)
        {
            if (_particles[i].IsActive)
            {
                // Actualizar física y modificadores
                _particles[i].Update(deltaTime);
                
                // NUEVO: Actualizar animación
                _particles[i].UpdateAnimation(deltaTime);
            }
        }
    }

    public Particle[] Particles => _particles;
}
```

```csharp
protected override void LoadContent()

    explosionProfile.Shape = new PointShape(5f);
    explosionProfile.Lifetime = new FloatRange(0.5f, 0.5f); // 0.5s = 16 frames a 30fps
    explosionProfile.Speed = new FloatRange(50, 150);
    explosionProfile.DirectionMode = EmissionDirectionMode.Random;
    
    // Configuración de animación
    explosionProfile.AnimateParticles = true;
    explosionProfile.Animation = explosionAnimation;
    explosionProfile.RandomStartFrame = false; // Todos empiezan desde el frame 0

    // Modificadores
    explosionProfile.AddModifier(new ScaleOverLifetimeModifier(0.5f, 2f));
    explosionProfile.AddModifier(new AlphaOverLifetimeModifier(1f, 0f));

    _emitter = new ParticleEmitter(Vector2.Zero, explosionProfile, _random);
    _renderer = new ParticleRenderer(GraphicsDevice, 10000);
}

protected override void Update(GameTime gameTime)
{
    _emitter.Update(gameTime, _particleSystem);
    _particleSystem.Update(gameTime);
}

protected override void Draw(GameTime gameTime)
{
    GraphicsDevice.Clear(Color.Black);

    // El renderer ahora usa UVs animadas automáticamente
    _renderer.UpdateVertexBuffer(_particleSystem.Particles, _explosionSheet);
    _renderer.Render(_explosionSheet, _projectionMatrix);
}
```

### C. Ping-Pong Animation (Ida y Vuelta)

```csharp
public class SpriteAnimation
{
    public bool PingPong { get; set; } = false;

    public int GetFrameForNormalizedAge(float normalizedAge)
    {
        if (!PingPong)
        {
            return (int)(normalizedAge * TotalFrames) % TotalFrames;
        }
        else
        {
            // Ida y vuelta
            float pingPongCycle = normalizedAge * 2;
            if (pingPongCycle > 1)
                pingPongCycle = 2 - pingPongCycle;
            
            return (int)(pingPongCycle * (TotalFrames - 1));
        }
    }
}
```

## 1. Estructura de Datos Orientada a Datos (SoA - Structure of Arrays)

El cambio **MÁS IMPORTANTE** para rendimiento CPU. En lugar de almacenar datos por partícula (AoS), almacena por propiedad (SoA). Esto mejora drásticamente la caché de la CPU.

### ❌ Antes (AoS - Array of Structures)
```csharp
public class Particle
{
    public Vector2 Position;
    public Vector2 Velocity;
    public Color Color;
    public float Scale;
    public bool IsActive;
}
Particle[] particles = new Particle[10000];
```
**Problema:** Al actualizar solo posiciones, cargas datos innecesarios (Color, Scale) en caché.

### ✅ Ahora (SoA - Structure of Arrays)
```csharp
public class ParticleSystem
{
    public Vector2[] Positions;
    public Vector2[] Velocities;
    public int[] Colors; // RGBA empaquetado
    public float[] Scales;
    public bool[] IsActive;
    public int ActiveCount;

    public ParticleSystem(int maxParticles)
    {
        Positions = new Vector2[maxParticles];
        Velocities = new Vector2[maxParticles];
        Colors = new int[maxParticles];
        Scales = new float[maxParticles];
        IsActive = new bool[maxParticles];
    }

    public void Update(float deltaTime)
    {
        // Acceso secuencial = mejor caché
        for (int i = 0; i < ActiveCount; i++)
        {
            if (!IsActive[i]) continue;
            
            Positions[i] += Velocities[i] * deltaTime;
            // Solo accedes a lo que necesitas
        }
    }
}
```

**Beneficio:** **2-5x más rápido** en actualizaciones CPU por mejor uso de caché L1/L2.

## 2. Culling por Distancia y Frustum

No actualices ni dibujes partículas que no se ven.

```csharp
public class ParticleCuller
{
    private float _maxDrawDistance;
    private Rectangle _viewport;

    public ParticleCuller(float maxDrawDistance, Rectangle viewport)
    {
        _maxDrawDistance = maxDrawDistance;
        _viewport = viewport;
    }

    public bool ShouldRender(Vector2 position, Vector2 cameraPosition)
    {
        // 1. Culling por distancia
        float distance = Vector2.Distance(position, cameraPosition);
        if (distance > _maxDrawDistance) return false;

        // 2. Culling por viewport (con margen)
        Rectangle expandedViewport = new Rectangle(
            _viewport.X - 100,
            _viewport.Y - 100,
            _viewport.Width + 200,
            _viewport.Height + 200
        );

        return expandedViewport.Contains((int)position.X, (int)position.Y);
    }
}

// En ParticleSystem.Update
for (int i = 0; i < ActiveCount; i++)
{
    if (!_culler.ShouldRender(Positions[i], _cameraPosition))
    {
        IsActive[i] = false; // Desactivar temprano
        continue;
    }
    // ... actualizar
}
```

**Beneficio:** Reduce carga de GPU y CPU en escenas grandes.

## 3. Level of Detail (LOD) para Partículas

Reduce la complejidad de partículas lejanas.

```csharp
public enum ParticleLOD
{
    Full,      // Textura animada, rotación, todos los modificadores
    Medium,    // Textura estática, sin rotación
    Low        // Solo punto de color (1 vértice)
}

public class ParticleSystem
{
    public ParticleLOD GetCurrentLOD(Vector2 position, Vector2 cameraPosition)
    {
        float distance = Vector2.Distance(position, cameraPosition);
        
        if (distance < 200) return ParticleLOD.Full;
        if (distance < 500) return ParticleLOD.Medium;
        return ParticleLOD.Low;
    }

    public void UpdateWithLOD(float deltaTime, Vector2 cameraPosition)
    {
        for (int i = 0; i < ActiveCount; i++)
        {
            var lod = GetCurrentLOD(Positions[i], cameraPosition);
            
            if (lod == ParticleLOD.Low)
            {
                // Solo actualizar posición, skip modificadores caros
                Positions[i] += Velocities[i] * deltaTime;
                continue;
            }
            
            // LOD Full/Medium: actualizar todo
            UpdateFullParticle(i, deltaTime);
        }
    }
}
```

**Beneficio:** **30-50% menos vértices** en escenas con partículas lejanas.

## 4. Parallel Processing (Task Parallel Library)

Divide la actualización de partículas entre múltiples núcleos de CPU.

```csharp
using System.Threading.Tasks;

public void UpdateParallel(float deltaTime)
{
    int batchSize = ActiveCount / Environment.ProcessorCount;
    
    Parallel.For(0, Environment.ProcessorCount, threadIndex =>
    {
        int start = threadIndex * batchSize;
        int end = (threadIndex == Environment.ProcessorCount - 1) 
            ? ActiveCount 
            : (threadIndex + 1) * batchSize;

        for (int i = start; i < end; i++)
        {
            if (!IsActive[i]) continue;
            
            Positions[i] += Velocities[i] * deltaTime;
            // ... otros updates
        }
    });
}
```

**⚠️ Advertencia:** Solo vale la pena con **10,000+ partículas**. El overhead de threading puede ser contraproducente para menos.

**Beneficio:** **2-4x más rápido** en CPUs multi-core (4+ núcleos).

## 5. Particle Budget System

Limita el número de partículas activas por frame para mantener FPS estables.

```csharp
public class ParticleBudget

    }

        _particlesEmittedThisFrame += count;
    }
}

// En ParticleEmitter
public void Update(GameTime gameTime, ParticleSystem system)

    }
}
```

**Beneficio:** Previene **frame spikes** cuando hay muchas explosiones simultáneas.

## 8. Memory Pooling Avanzado

Evita GC allocs en caliente.

```csharp
public class ParticlePool
{
    private Particle[] _pool;
    private Stack<int> _availableIndices;
    private int _maxSize;

    public ParticlePool(int maxSize)
    {
        _maxSize = maxSize;
        _pool = new Particle[maxSize];
        _availableIndices = new Stack<int>(maxSize);
        
        for (int i = 0; i < maxSize; i++)
        {
            _pool[i] = new Particle();
            _availableIndices.Push(i);
        }
    }

    public Particle Get()
    {
        if (_availableIndices.Count == 0) return null;
        int index = _availableIndices.Pop();
        _pool[index].IsActive = true;
        return _pool[index];
    }

    public void Return(Particle particle)
    {
        particle.IsActive = false;
        int index = Array.IndexOf(_pool, particle);
        _availableIndices.Push(index);
    }
}
```

**Beneficio:** **0 allocs por frame** = sin GC pauses.
